# NeuroShield Dashboard - Real Data + Simulation Guide

## What Changed

### 1. **Real Data Integration**
- **Backend**: Added new `/api/dashboard/system-metrics` endpoint serving 96 realistic 24-hour metrics
- **Metrics Include**: CPU, memory, error rates, response times, pod restarts
- **Data Source**: `data/system_metrics.json` generated by `generate_realistic_scenario.py`

### 2. **Realistic Simulation**
- **Smart Generation**: Incidents now generated based on actual system state
- **Rules**:
  - CPU > 85% → `scale_up` (60% probability)
  - Memory > 85% → `restart_pod` (50% probability)
  - Error rate > 5% → `retry_build` (70% probability)
  - Pod restarts > 3 → `rollback_deploy` (40% probability)
- **Healing Success**: 90% of incidents self-heal automatically

### 3. **Currency Changed**
- Changed from USD (`$`) to Indian Rupees (`₹`)
- Cost saved displayed in Rs

### 4. **Dashboard Controls**
- **Start/Stop Simulation Button**: Green = active, Amber = stopped
- **Real-time Updates**: Every 5 seconds from API
- **Simulation Intervals**: Every 3 seconds when enabled

## How to Use

### Start Simulation
```bash
./scripts/start_simulation.sh
```

This will:
1. Generate realistic 24-hour scenario (26 healing events, 92.3% success rate)
2. Verify backend services
3. Open dashboard at http://localhost:5173

### Stop Simulation
```bash
./scripts/stop_simulation.sh
```

## Real vs Simulated Incidents

### Real Incidents (From Orchestrator)
- Source: `data/healing_log.json` (Written by orchestrator)
- Represents actual CI/CD failures detected
- Mixed with simulation for continuous demo

### Simulated Incidents (From Dashboard)
- Generated client-side when simulation enabled
- Based on realistic system metrics
- Demonstrates healing response patterns
