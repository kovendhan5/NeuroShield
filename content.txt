üéØ WHAT IS THIS PROJECT?
================================================================================

NeuroShield is an AIOps (Artificial Intelligence for IT Operations) system that 
automatically detects and fixes CI/CD pipeline failures. Instead of waiting for 
someone to manually troubleshoot when a build fails, this system:

1. PREDICTS failures before they happen (using AI)
2. AUTOMATICALLY fixes them (using reinforcement learning)
3. REDUCES downtime by ~40%+ (compared to manual fixing)

Think of it like having a smart robot watching your build pipelines 24/7, learning 
from past failures, and automatically fixing problems like:
- Out of memory errors
- Flaky tests
- Dependency issues
- Resource constraints

================================================================================
üèóÔ∏è PROJECT ARCHITECTURE
================================================================================

The system has 4 main components:

1. TELEMETRY COLLECTOR (‚úÖ COMPLETED)
   - Polls Jenkins API every 10 seconds
   - Collects Prometheus metrics (CPU, memory, pods)
   - Saves everything to CSV for training the AI
   - Location: src/telemetry/

2. FAILURE PREDICTOR (üîÑ IN PROGRESS - Week 2)
   - Uses DistilBERT transformer model (AI for text analysis)
   - Reads build logs and predicts if failure is coming
   - Target: 80%+ accuracy
   - Location: src/prediction/

3. RL AGENT (üìÖ PLANNED - Week 3)
   - Uses PPO (Proximal Policy Optimization) reinforcement learning
   - Decides what action to take when failure detected
   - 4 Actions: Retry build, Scale pods, Rollback, Do nothing
   - Location: src/rl_agent/

4. ORCHESTRATOR (üìÖ PLANNED - Week 3-4)
   - Executes the actions decided by RL agent
   - Calls Jenkins API to retry builds
   - Uses kubectl to scale Kubernetes pods
   - Includes Streamlit dashboard for monitoring
   - Location: src/orchestration/

üéì RESEARCH BACKGROUND
================================================================================

This project is inspired by a research paper: "AIOps-Driven Self-Healing 
Pipelines" (Kovendhan P et al., 2025)

Key findings from the paper:
- 47% reduction in Mean Time To Recovery (MTTR)
- 92% prediction accuracy
- 6 autonomous recovery actions
- Tested on Sock Shop microservices benchmark

NeuroShield is implementing a simplified version with:
- 4 actions instead of 6
- Target: 40% MTTR reduction (vs 47% in paper)
- Target: 80% accuracy (vs 92% in paper)
- MVP for portfolio demonstration

üí° HOW THE SYSTEM WORKS (End-to-End Flow)
================================================================================

1. CONTINUOUS MONITORING
   ‚îî‚îÄ> Telemetry collector polls Jenkins & Prometheus every 10 seconds

2. FAILURE DETECTION
   ‚îî‚îÄ> AI model analyzes logs and metrics in real-time
   ‚îî‚îÄ> Predicts: "This build is about to fail!"

3. STATE GENERATION
   ‚îî‚îÄ> System creates state vector (~20 dimensions)
   ‚îî‚îÄ> Includes: pipeline health, resource usage, log embeddings

4. ACTION SELECTION
   ‚îî‚îÄ> RL Agent evaluates state
   ‚îî‚îÄ> Chooses best action:
       ‚Ä¢ Retry stage
       ‚Ä¢ Scale pods (+20%)
       ‚Ä¢ Rollback to previous version
       ‚Ä¢ Do nothing (if false alarm)

5. ACTION EXECUTION
   ‚îî‚îÄ> Orchestrator executes via APIs:
       ‚Ä¢ Jenkins REST API for retries/rollbacks
       ‚Ä¢ kubectl commands for pod scaling

6. FEEDBACK & LEARNING
   ‚îî‚îÄ> System measures outcome (success/failure)
   ‚îî‚îÄ> RL agent learns from result
   ‚îî‚îÄ> Improves future decisions
