SYSTEM ARCHITECTURE
═════════════════════════════════════════════════════════════════════

  Jenkins/Prometheus → TelemetryCollector
       ↓
  Build 52D State Vector
       ↓
  DistilBERT Log Encoder (768D) → PCA (16D)
       ↓
  FailurePredictor Neural Network → probability
       ↓
  PPO RL Agent (Stable Baselines3) → 1 of 6 actions
       ↓
  Rule-Based Overrides (CPU/Memory/Restart checks)
       ↓
  Execute Healing Action:
    • Action 0: restart_pod (kill + recreate)
    • Action 1: scale_up (increase replicas)
    • Action 2: retry_build (re-run Jenkins)
    • Action 3: rollback_deploy (revert version)
    • Action 4: clear_cache (flush caches)
    • Action 5: escalate_to_human (alert + report)
       ↓
  Log Results → Dashboard Update


EXPECTED BEHAVIOR
═════════════════════════════════════════════════════════════════════

Normal Cycle (no failures):
  Probability: 10-25%
  Action: Usually 0 (restart_pod) or skip
  MTTR: Baseline (no action needed)

Anomaly Detected (high resource use):
  Probability: 60-85%
  Action: 1 (scale_up) or 4 (clear_cache)
  MTTR: Improvements 20-40%

Critical Failure:
  Probability: 90%+
  Action: 5 (escalate_to_human)

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════

Q: Orchestrator won't start
A: Check if Jenkins is running (curl http://localhost:8080)
   Check logs: tail -f logs/orchestrator.log

Q: Dashboard shows old data
A: Refresh browser (Ctrl+R)
   Check if orchestrator is polling (see console)

Q: Port already in use
A: Jenkins (8080): netstat -ano | findstr :8080
   Prometheus (9090): netstat -ano | findstr :9090
   Dashboard (8501): python -m streamlit run app.py --server.port 8502

Q: Models not found
A: Models should be in models/ directory
   If missing: python src/prediction/train.py

Q: No telemetry updates
A: Check data/healing_log.json is growing
   Check orchestrator shows "Cycle N" messages
